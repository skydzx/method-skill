# Cases

这组题故意覆盖 4 类任务：

- 社会分析
- 组织诊断
- 技术 / AI
- 观点批判

每题后面的“最低命中点”不是标准答案，而是最低限度的方法信号。

## Social Analysis

### S1

**Prompt**

> Use $method-skill to analyze this problem: 为什么很多年轻人明明反感内卷，却还是不断把自己卷进去？

**最低命中点**

- 不能只归因于个人心态
- 要提到就业、教育筛选、生活成本、竞争结构等现实条件
- 要指出上升压力与有限稳定位置之间的核心张力
- 结论不能停在“放松心态”

### S2

**Prompt**

> Use $method-skill to analyze this problem: 为什么一些城市不断修建商业综合体，但很多普通人还是觉得公共生活越来越贫乏？

**最低命中点**

- 区分消费空间和公共空间
- 提到土地、资本回报、城市治理、公共资源配置
- 不能把问题只理解成“审美变差”
- 要给出结构性解释

### S3

**Prompt**

> Use $method-skill to analyze this problem: 为什么平台经济一边提高便利性，一边又让劳动者更不稳定？

**最低命中点**

- 同时看到效率提升和劳动不稳定
- 提到平台控制、算法管理、成本转移、劳动关系重组
- 要抓“便利性扩张”与“风险外包”之间的张力

## Organization Diagnosis

### O1

**Prompt**

> Use $method-skill to analyze this problem: 为什么有些公司天天开会，但效率还是很低？

**最低命中点**

- 不能只说执行力差
- 要提到职责不清、决策权分散、信息流混乱、机制代偿
- 要指出会议过多是症状，不是根因

### O2

**Prompt**

> Use $method-skill to diagnose this problem: 为什么一个团队明明招了更多人，推进速度却没有明显变快？

**最低命中点**

- 不能把“人多”自动等于“效率高”
- 要提到协作成本、流程瓶颈、责任边界、管理复杂度
- 要指出主要矛盾是什么，而不是列一堆可能性

### O3

**Prompt**

> Use $method-skill to diagnose this problem: 为什么很多组织总在追求流程规范，结果一线执行反而越来越僵硬？

**最低命中点**

- 区分规范化的目标和实际效果
- 提到控制需求、责任转移、反馈机制不足、上下游脱节
- 不能停留在“员工不够灵活”

## Technology and AI

### T1

**Prompt**

> Use $method-skill to analyze this problem: AI 会不会自动把社会带向更公平的方向？

**最低命中点**

- 反对技术决定论
- 提到算力、数据、模型、所有权、分配
- 要指出技术方向受制度和控制权影响

### T2

**Prompt**

> Use $method-skill to analyze this problem: 为什么生成式 AI 一边提升效率，一边又让很多知识工作者感到更不安全？

**最低命中点**

- 同时分析生产率和劳动不安全
- 提到劳动重组、岗位贬值、平台集中、收益分配
- 不能只说“技术进步总会淘汰一部分人”

## Critique

### C1

**Prompt**

> Use $method-skill to critique this statement: “只要努力，普通人也一定能成功。”

**最低命中点**

- 不能只从道德上说这句话残酷
- 要指出它把结构问题改写成个人努力问题
- 要提到资源分布、起点差异、机会结构、筛选机制

### C2

**Prompt**

> Use $method-skill to critique this statement: “技术本身是中立的，关键只看使用它的人。”

**最低命中点**

- 不能只简单回答“对/不对”
- 要指出技术嵌在所有权、制度、组织和使用场景中
- 要说明“中立”说法怎样遮蔽了控制关系和社会后果
