很多 AI 的问题，不是不会说，而是说得太快、太满、太表面：

- 一上来就下判断，没有先界定对象
- 一上来就谈价值，没有先看现实条件
- 把现象当本质，把症状当根因
- 把一堆问题摊平，抓不住主要矛盾
- 会写结论，却不会把问题放进历史过程和实践条件里

`method-skill` 的目标，就是把这些毛病压下去。

它不是一个“哲学概念包”，也不是一个“辩论话术包”。  
它是一套给 AI 使用的工作方法：先看对象，再看条件；先抓矛盾，再看历史；从特殊上升到一般，再回到实践。

## 这是什么

`method-skill` 是一个给 Codex / Claude Code 一类 AI agent 使用的 skill。

它会约束 AI 优先做这些事情：

1. 先界定对象
2. 再看物质条件和现实约束
3. 再抓主要矛盾
4. 再放到历史过程中理解
5. 再从特殊上升到一般
6. 最后回到实践、条件与可操作结论

它尤其适合这些任务：

- 社会问题分析
- 制度与组织诊断
- 观点批判与叙事拆解
- 战略建议
- AI、平台、自动化等技术问题
- 需要从“现实条件”出发的中文分析写作

## 安装

把本仓库作为一个 skill 目录放到你的 skills 目录下，或按你所用 agent 的 skill 机制引入。

如果你的环境支持直接按仓库加载 skill，就把这个仓库作为独立 skill 仓库使用。

## 使用方式

引入后，可以直接这样调用：

```text
Use $method-skill to analyze this problem.
用 $method-skill 分析这个问题。
用方法论.skill 来拆解这个社会现象。
```

## 核心方法

这套 skill 的方法主线很简单，但足够硬：

- 从具体对象出发，不先套理论标签
- 从现实条件出发，不从口号和愿望出发
- 先分主次矛盾，不把问题摊平成一张清单
- 把对象放进历史过程，而不是静止理解
- 从特殊上升到一般，再回到具体校验
- 最后落到实践含义，而不是停在抽象表态

如果问题涉及制度、文化、技术或政治经济，它还会继续追问：

- 谁拥有
- 谁控制
- 谁获益
- 谁承担代价
- 什么在维持现状
- 什么在推动变化

## 示例问题

- AI 会不会让我们走向共产主义社会？
- 为什么很多组织天天开会但效率还是低？
- 如何批判“只要努力就能成功”这种说法？
- 用现实条件、主要矛盾和历史过程分析某个政策问题
- 用方法论而不是空话去分析一个平台、组织或制度问题

## 效果展示

### 示例 1：社会分析

**问题**

> 为什么很多年轻人明明反感内卷，却还是不断把自己卷进去？

**method-skill 的回答**

先把问题说清楚。这里不是单纯的“年轻人想不开”，而是竞争压力、就业结构、教育筛选和生活成本共同作用的结果。  
表面上看是个人选择，更深一层是人在不安全的现实条件下，只能通过不断加码来换取相对位置。  
这里的主要矛盾，不是“想躺平”和“想成功”的心理冲突，而是越来越高的生存与上升压力，和越来越有限的稳定位置之间的矛盾。  
所以关键不在于劝人想开一点，而在于看是什么制度和分配结构在持续制造这种竞争。

### 示例 2：组织诊断

**问题**

> 为什么有些公司天天开会，但效率还是很低？

**method-skill 的回答**

先不要把问题理解成“员工执行力差”或“管理者不够努力”。  
很多会议过多的组织，真实问题往往是职责不清、决策权分散、信息流混乱，以及管理层对过程缺乏稳定控制。  
会议在这里不是解决问题的手段，而是在代偿结构问题：因为平时机制不清，所以只能不断临时协调。  
真正要抓的主要矛盾，不是“会开得太多”，而是组织结构和责任机制无法支撑高效协作。  
只要这一点不改，减少几场会通常也只是表面修补。

### 示例 3：技术与 AI

**问题**

> AI 会不会自动把社会带向更公平的方向？

**method-skill 的回答**

不能把 AI 当成自动推动历史的主体。  
AI 首先是一种新的生产力，但它会朝哪个方向发展，不取决于技术本身，而取决于谁拥有算力、数据、模型和分配权。  
如果 AI 主要被平台和资本控制，它更可能强化垄断、监控和收益集中，而不是自动扩大公共利益。  
这里的关键，不是 AI 够不够先进，而是社会化生产能力和私人占有之间的矛盾会不会被重新组织。  
所以问题不该问“AI 会不会自动带来公平”，而应问“在什么所有制和治理结构下，AI 才可能服务更广泛的人”。

## 评测

如果你不想只凭感觉判断这个 skill 是否有效，可以直接使用仓库里的最小评测集：

- `evals/README.md`
- `evals/cases.md`

它提供了 10 个固定问题，以及 5 个最关键的评分维度：

1. 是否先界定对象
2. 是否回到现实条件
3. 是否抓到主要矛盾
4. 是否有历史 / 过程意识
5. 是否回到实践含义

这一步的意义不是把回答打分打得很学术，而是稳定发现：模型是不是又滑回了空话、道德化和表面化。

## 适合什么，不适合什么

适合：

- 需要结构分析、现实分析、历史分析的问题
- 需要判断“表面现象后面真正起作用的机制”的问题
- 需要把 AI 从空洞总结拉回现实约束和实践条件的任务

不适合：

- 只想要一句立场表态
- 只想堆砌哲学名词
- 只想做纯修辞性输出而不关心解释力

## 一句话总结

如果你想让 AI 少一点空话、少一点唯心、少一点表面化，多一点现实感、结构感、历史感和实践感，这个 skill 就是为这个目的写的。
